# 마이페이지 확장 계획

## 목표
설정 탭을 "MY" 탭으로 확장하여 내 판사, 내 판결문 관리 기능 제공.
기존에 API만 있고 UI가 없던 기능들을 연결.

## 변경사항

### 1. BottomNav 변경
- 설정(Settings) 탭 -> MY 탭 (아이콘: User)
- 경로: `/settings/` -> `/my/`

### 2. MY 페이지 (신규: `/my/`)
마이페이지 허브. 카드형 메뉴로 각 섹션 이동.
- 프로필 카드 (닉네임, 토큰 잔액, 충전)
- 내 판결문 (`/my/fights/`) - 바로가기 카드
- 내 판사 (`/my/judges/`) - 바로가기 카드
- 테마 선택 (인라인)
- 로그아웃 / 회원탈퇴 / 앱 정보

### 3. 내 판결문 페이지 (신규: `/my/fights/`)
기존 `/history/` 내용을 이동. 필터/정렬 강화.
- 전체 / 승소 / 패소 탭
- 카드에 판사명, 날짜, 잘못비율 표시
- 카드 클릭 -> `/fight/?id=`

### 4. 내 판사 페이지 (신규: `/my/judges/`)
내가 만든 판사 목록 관리.
- 판사 카드 (이름, 설명, 사용횟수, 점수)
- 승인 상태 뱃지 (대기중/승인/반려)
- 판사 만들기 FAB -> `/judges/create/`

### 5. 판사 상세 페이지 (신규: `/judges/[id]/`)
- 판사 정보 (이름, 설명, 사용횟수, 점수)
- 투표 UI (좋아요/싫어요)
- "이 판사로 판결받기" CTA

### 6. 기존 페이지 정리
- `/history/` -> `/my/fights/`로 리다이렉트
- `/settings/` -> `/my/`로 리다이렉트
- BottomNav 기록 탭 제거 (MY 안에 통합)

## BottomNav 최종
| 탭 | 아이콘 | 경로 |
|----|--------|------|
| 홈 | Home | `/` |
| 판사 | Gavel | `/judges/` |
| MY | User | `/my/` |

3탭으로 정리. 기록은 MY 안에 통합.

## 작업 순서
1. `/my/` 페이지 생성
2. `/my/fights/` 페이지 생성
3. `/my/judges/` 페이지 생성
4. `/judges/[id]/` 판사 상세 페이지 생성
5. BottomNav 3탭으로 변경
6. 기존 `/history/`, `/settings/` 리다이렉트 처리
7. 빌드 확인
