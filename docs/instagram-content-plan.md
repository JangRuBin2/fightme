# FightMe Instagram Content Plan

> "AI로 앱 만드는 개발자" 계정 컨셉
> 토스 미니앱 + AI + 1인 개발 경험 공유

## Series 1: "AI한테 코딩 시켰더니" (개발 과정)

바이럴 가능성 높은 메인 시리즈. 개발자 + 비개발자 모두 흥미 느낄 수 있는 포맷.

### 1-1. "AI한테 앱 만들어달라고 했더니 진짜 만들어줌"

- **포맷**: 릴스 (화면녹화 타임랩스)
- **내용**: Claude Code 터미널에서 코드가 자동으로 써지는 과정. 빈 폴더 -> 동작하는 앱까지
- **촬영**: 터미널 화면녹화 + 중간중간 앱 실행 화면 삽입
- **예상 반응**: "이게 된다고?" / "개발자 실직 각"
- **난이도**: 낮음 (화면녹화만 있으면 됨)

### 1-2. "내가 코드 1줄도 안 쓴 페이지가 있다"

- **포맷**: 카루셀 (5-7장)
- **내용**:
  - 장 1: 후킹 - "이 앱에는 내가 코드 1줄도 안 쓴 페이지가 있습니다"
  - 장 2: 실제 페이지 스크린샷
  - 장 3: AI가 생성한 코드 하이라이트
  - 장 4: 내가 수정한 부분 (비교)
  - 장 5: AI 코딩의 현실적 비율 - "AI 80% + 내 수정 20%"
- **난이도**: 낮음

### 1-3. "AI한테 버그 고치라고 했더니 3초 만에..."

- **포맷**: 릴스 (15-30초)
- **내용**: 빌드 에러 화면 -> Claude Code에 에러 붙여넣기 -> 자동 수정 -> 빌드 성공
- **포인트**: 속도감 있는 편집. 에러 빨간 글씨 -> 초록 성공 메시지 전환
- **난이도**: 낮음

### 1-4. "AI 개발의 현실 (feat. 삽질 모음)"

- **포맷**: 카루셀 (7-10장)
- **내용**:
  - AI가 엉뚱하게 이해한 순간 스크린샷
  - 같은 실수 반복하는 장면
  - 되돌리기(undo) 연속 사용
  - "분명 고쳤다더니 다른 데가 깨짐"
  - 마지막 장: "그래도 혼자보다 10배 빠름"
- **포인트**: 현실적인 면을 보여주되, 결론은 긍정적으로
- **난이도**: 중간 (과정에서 스크린샷 모아둬야 함)

## Series 2: "토스 미니앱 만들기" (기술 공유)

개발자 타겟. 토스 앱인토스 관련 정보가 거의 없어서 검색 유입 기대.

### 2-1. "토스 미니앱, 아무도 안 알려주는 것들"

- **포맷**: 카루셀 (5-7장)
- **내용**:
  - `document.write()` 쓰면 앱 죽음
  - Static Export 강제 (SSR 불가)
  - WebView 제약 (다운로드 불가 등)
  - mTLS 인증서 설정의 고통
  - SPA 라우팅을 위한 해킹
- **디자인**: 코드 스니펫 + 경고 아이콘 스타일
- **난이도**: 낮음

### 2-2. "Next.js로 토스 미니앱 만들 때 주의점 5가지"

- **포맷**: 카루셀 (6장)
- **내용**:
  1. `output: 'export'` 필수 - API Routes 사용 불가
  2. `images: { unoptimized: true }` - Next.js Image 최적화 불가
  3. SPA fallback용 index.html 후처리 스크립트 필요
  4. 모든 페이지에 `'use client'` 지시어
  5. `trailingSlash: true` 설정
- **난이도**: 낮음

### 2-3. "Supabase + 토스 로그인 연동 구조"

- **포맷**: 카루셀 (5장)
- **내용**:
  - 장 1: 전체 플로우 다이어그램 (토스SDK -> Edge Function -> Supabase Auth)
  - 장 2: mTLS 인증서로 토스 API 호출하는 구조
  - 장 3: HMAC-SHA256으로 비밀번호 생성하는 패턴
  - 장 4: RLS(Row Level Security)로 데이터 격리
  - 장 5: 이 구조의 장단점 정리
- **난이도**: 중간 (다이어그램 제작 필요)

### 2-4. ".ait 파일이 뭔지 아는 사람?"

- **포맷**: 릴스 (30초)
- **내용**: `pnpm build` -> `pnpm build:ait` -> .ait 파일 생성 -> 토스 개발자센터 업로드
- **포인트**: "앱스토어 심사 대신 토스 심사를 받습니다" 자막
- **난이도**: 낮음

## Series 3: "AI 판사 만들기" (기획/재미)

비개발자도 볼 수 있는 재미 콘텐츠. 앱 자체 홍보 효과.

### 3-1. "AI 판사한테 실제 커플 싸움 넣어봤다"

- **포맷**: 릴스 또는 카루셀
- **내용**:
  - 시나리오 1: "이불 빨래 누가 해야 됨?"
  - 시나리오 2: "카톡 읽씹 vs 바쁘다는 핑계"
  - 시나리오 3: "기념일 깜빡한 건 죄인가"
  - 각 시나리오별 AI 판결 + 과실 비율 공개
- **포인트**: 댓글에서 "나도 넣어보고 싶다" 반응 유도
- **난이도**: 낮음 (앱 스크린샷만 있으면 됨)

### 3-2. "판사 성격 바꾸면 판결이 달라질까?"

- **포맷**: 카루셀 (6장)
- **내용**:
  - 장 1: 같은 싸움 내용 제시
  - 장 2-4: 판사별 판결 비교
    - 냉철한 판사: "감정은 배제하고..."
    - 다정한 판사: "서로의 마음을 이해하면..."
    - 웃긴 판사: "둘 다 웃기네요..."
  - 장 5: 과실 비율 비교 그래프
  - 장 6: "어떤 판사가 제일 맞는 것 같아요?" (투표 유도)
- **난이도**: 낮음

### 3-3. "친구들한테 써보라고 했더니 반응이..."

- **포맷**: 릴스 (리액션 영상)
- **내용**: 실제 지인 사용 장면 촬영 (허락 필수)
- **포인트**: 판결 결과 보고 놀라거나 웃는 자연스러운 반응
- **난이도**: 중간 (촬영 필요)

### 3-4. "유저가 만든 판사 중 제일 웃긴 거"

- **포맷**: 카루셀
- **내용**: 유저 생성 판사들 중 독특한 캐릭터 소개
- **시기**: 런칭 후 유저 데이터 쌓인 뒤
- **난이도**: 낮음

## Series 4: "1인 개발 일지" (공감/브랜딩)

개발자 커뮤니티 타겟. 꾸준한 팔로우 유도.

### 4-1. "퇴근 후 사이드 프로젝트 한 달 타임라인"

- **포맷**: 카루셀 (5장)
- **내용**:
  - Week 1: 기획 - 아이디어 메모, 와이어프레임
  - Week 2: MVP - 핵심 기능 구현 화면
  - Week 3: 디자인 삽질 - 테마 3개 만든 과정
  - Week 4: 배포 - 토스 심사, 최종 화면
  - 마지막: Before/After 비교
- **난이도**: 중간 (과정 스크린샷 필요)

### 4-2. "토스 심사 리얼 후기"

- **포맷**: 카루셀 (5-7장)
- **내용**:
  - 심사 제출 ~ 결과까지 타임라인
  - 리젝 사유 (있다면) + 수정 과정
  - 또는 심사 통과 팁 정리
  - 앱인토스 개발자센터 실제 화면
- **난이도**: 낮음

### 4-3. "사이드 프로젝트 비용 전부 공개"

- **포맷**: 카루셀 (4-5장)
- **내용**:
  - Supabase: 무료 플랜 (0원)
  - Google Gemini API: 월 예상 비용
  - 도메인/인프라: 비용 내역
  - Claude Code: 구독 비용
  - 시간 투자: 총 N시간 (가치 환산)
  - 합계 + "커피 N잔 값으로 앱 만든 셈"
- **난이도**: 낮음

### 4-4. "다 만들고 나서 아무도 안 쓰면?"

- **포맷**: 카루셀 또는 릴스
- **내용**:
  - 런칭 후 실제 DAU, 설치 수 공개 (솔직하게)
  - 예상 vs 현실 비교
  - 그래도 얻은 것들 (기술 경험, 포트폴리오, 콘텐츠)
  - "사이드 프로젝트의 진짜 가치"
- **시기**: 런칭 2-4주 후
- **난이도**: 낮음

### 콘텐츠 제작 팁

- **화면녹화는 미리미리**: 개발하면서 터미널 화면, 에러 화면, 성공 화면 수시로 녹화
- **Before/After 스크린샷**: 기능 추가할 때마다 전/후 캡처
- **에러 메시지 수집**: 재밌거나 황당한 에러는 바로 스크린샷
- **카루셀 디자인**: Canva 또는 Figma 템플릿 하나 만들어두면 빠름
- **릴스 길이**: 15-30초가 최적. 60초 넘기지 않기
- **CTA**: 매 게시물 마지막에 "저장해두세요" 또는 "이런 경험 있으면 댓글로"
